# 資料擷取與處理管線

## 狀態

draft

## 概述

建立自動化資料管線，從台灣政府公開資料平台（data.gov.tw）擷取資料集，經過清洗與轉換後儲存為結構化格式，供 API 服務與視覺化前端使用。

## 需求

### 資料擷取

- 支援從 data.gov.tw 的 API 擷取 JSON/CSV 格式資料
- 支援設定擷取排程（定時更新）
- 記錄每次擷取的狀態與時間戳

### 資料清洗與轉換

- 處理缺失值、異常值
- 統一日期格式、數值格式、地區名稱
- 將原始資料轉換為標準化的 JSON 格式

### 資料儲存

- 處理後的資料以 JSON 檔案儲存於 `data/` 目錄
- 保留原始資料備份於 `data/raw/` 目錄
- 提供資料版本管理機制

## 非目標

- 不建立自有資料庫系統（初期以檔案儲存為主）
- 不處理需要登入才能取得的非公開資料
- 不實作即時串流資料處理

## 技術方案

- 使用 Python `requests` 擷取資料
- 使用 `Pandas` 進行資料清洗與轉換
- 使用 `logging` 記錄處理日誌

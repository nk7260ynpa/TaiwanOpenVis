## ADDED Requirements

### Requirement: 首頁

系統 SHALL 提供首頁（`/`），展示專案簡介與精選視覺化入口。首頁 MUST 包含前往地圖頁面的導引連結。

#### Scenario: 使用者進入首頁

- **WHEN** 使用者開啟網站根路徑 `/`
- **THEN** 頁面顯示專案標題「TaiwanOpenVis — 台灣公開資料視覺化」、簡介文字、前往地圖頁的按鈕

### Requirement: 導覽列

系統 SHALL 在所有頁面頂部顯示固定的導覽列，包含網站標題與頁面連結。

#### Scenario: 導覽列顯示於所有頁面

- **WHEN** 使用者開啟任何頁面
- **THEN** 頁面頂部顯示導覽列，包含「首頁」與「人口地圖」連結

#### Scenario: 導覽列響應式收合

- **WHEN** 瀏覽器寬度小於 768px
- **THEN** 導覽列的連結收合為漢堡選單按鈕，點擊後展開

### Requirement: 資料詳情頁

系統 SHALL 提供資料詳情頁（`/detail/{county}`），以表格與統計摘要呈現特定縣市的人口資料。

#### Scenario: 從地圖頁導向詳情頁

- **WHEN** 使用者在鄉鎮視圖中點擊「查看詳情」連結
- **THEN** 導向至該縣市的詳情頁，頁面顯示縣市名稱、鄉鎮人口表格（含鄉鎮名稱、戶數、男性人口、女性人口、總人口）

#### Scenario: 詳情頁表格排序

- **WHEN** 使用者點擊表格欄位標題（如「總人口」）
- **THEN** 表格依該欄位升冪或降冪排序

### Requirement: 頁面共用樣式

系統 SHALL 使用統一的 CSS 樣式表，確保所有頁面的視覺風格一致。樣式 MUST 採用中性色調搭配藍綠色系作為主色調。

#### Scenario: 樣式一致性

- **WHEN** 使用者瀏覽不同頁面
- **THEN** 所有頁面共用相同的字體、配色、間距與排版風格

### Requirement: 暗色模式主題變數

系統 SHALL 在 CSS 中定義暗色主題的色彩變數，當 `[data-theme="dark"]` 屬性套用於 `<html>` 元素時，所有 CSS 變數 MUST 切換為暗色主題對應值。

#### Scenario: 暗色模式色彩套用

- **WHEN** `<html>` 元素的 `data-theme` 屬性設為 `"dark"`
- **THEN** 頁面背景、文字、卡片、導覽列等元件 SHALL 使用暗色主題配色

### Requirement: 導覽列主題切換按鈕

系統 SHALL 在導覽列中提供主題切換按鈕，允許使用者手動切換亮色/暗色模式。

#### Scenario: 點擊切換按鈕

- **WHEN** 使用者點擊導覽列中的主題切換按鈕
- **THEN** 頁面主題在亮色與暗色之間切換，按鈕圖示同步更新
